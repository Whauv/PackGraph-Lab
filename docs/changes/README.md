# Change Tracking

Use this folder to keep lightweight milestone notes so the repo is easier to follow over time.

## Suggested format

Create one markdown note per meaningful update, for example:

- `2026-07-21-layout-remap.md`
- `2026-07-21-neo4j-runtime.md`
- `2026-07-21-community-shell.md`

## What to include

- What changed
- Why it changed
- Main files touched
- Follow-up work or known gaps

## Template

```md
# Update Title

Date: YYYY-MM-DD

## Summary

Short description of the change.

## Main files

- `path/to/file`
- `path/to/another-file`

## Notes

- Important follow-up
- Risk or limitation
```
